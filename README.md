# EdOr

## One source. New combinations. Different finished results.

Turn your notes, documents, drafts, and ideas into finished work you can actually use—and do it again without starting over.

EdOr gives you a practical workbench for deciding who the AI should act as, how it should approach the work, and what it should do step by step. Change the Role, Mode, or Steps to produce something different from the same material. Keep a combination that works and run it again with new material to produce the same kind of result.

## From one source to many useful outcomes

A product brief does not have only one possible destination.

Use one EdOr workflow to turn it into a landing page. Use another to produce a technical specification. Change the setup again for a press release, an internal brief, a customer-facing explanation, or a campaign concept.

The source stays grounded. The workflow changes. The finished result changes with it.

## How EdOr works

### 1. Give EdOr the work

Start with the material you already have:

- notes;
- documents;
- drafts;
- transcripts;
- research;
- product information;
- source copy;
- or an undeveloped idea.

Add the surrounding context, the assignment, the desired outcome, and what a successful result should accomplish.

### 2. Shape the process

Choose the combination that fits the job:

- **Role** — who the AI should act as;
- **Mode** — how it should approach the work;
- **Steps** — what it should do, in what order.

A Role changes the point of view. A Mode changes the character of the work. Steps define the actual sequence of operations.

### 3. Run the sequence

EdOr executes the selected Steps in order. Each Step receives the current Material, performs its focused intervention, and hands the resulting Material to the next Step.

Run the sequence once for a direct transformation or use loops when the work benefits from another pass.

### 4. Inspect the work

The workbench keeps the process visible. Review the selected setup, each Step in the sequence, the Material entering and leaving it, the trace, and the final result.

You are not limited to a single opaque prompt or a result with no visible path behind it.

### 5. Use the workflow again

When a combination produces the kind of result you need, keep it. Replace the source Material and run the same process again.

You can also keep the Material and change the combination to produce a different finished result.

## Built from reusable parts

EdOr includes an editable library of Roles, Modes, and Steps.

### Roles

Roles define the specialist perspective brought to the work. A Role might operate as a writer, editor, copy architect, auditor, interpreter, translator, analyst, or another specialist you define.

### Modes

Modes define how the specialist approaches the assignment: creatively, analytically, correctively, persuasively, objectively, or through another working method suited to the task.

### Steps

Steps are focused operations. They may draft, expand, critique, consolidate, extract, translate, audit, restructure, compress, or perform another clearly defined intervention.

Roles, Modes, and Steps are independent, reusable parts. Add new ones, edit existing ones, and combine them without rebuilding the workbench.

- landing pages and website copy;
- articles, reports, and white papers;
- product briefs and technical material;

The system is not limited to one fixed workflow or one kind of output.

## A workbench, not a prompt collection

EdOr brings the full run into one place:

- structured source input;
- model selection;
- Role and Mode selection;
- ordered pre-Steps, Steps, and post-Steps;
- configurable loops;
- a live run summary;
- visible Step traces and handoffs;
- persistent run records;
- final Material ready to use;
- an editable library of workflow components.

The result is a repeatable working process rather than a folder of disconnected prompts.

## Packaged and production-ready

EdOr is delivered as a self-contained web workbench with:

- a browser-based interface;
- a Python orchestration engine;
- an editable filesystem-backed library;
- persistent run data;
- Docker packaging;
- support for local or hosted OpenAI-compatible model endpoints;
- environment-based model configuration;
- automated tests for library loading, sequencing, handoffs, and loops.

Run it on your own infrastructure, connect the model endpoint you want to use, and keep the workflow definitions and run records under your control.
